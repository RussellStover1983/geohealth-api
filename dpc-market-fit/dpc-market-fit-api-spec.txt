================================================================================
DPC MARKET FIT API — TECHNICAL SPECIFICATION
================================================================================
Build a FastAPI application that evaluates the geographic market viability for
Direct Primary Care (DPC) practices. The tool accepts a location (address, 
lat/lon, or ZIP code), resolves it to census tract(s), assembles data from 
multiple federal sources, and returns a composite DPC Market Fit Score with 
five dimensional sub-scores.

This follows the same architectural pattern as the SDOH Context API — 
location in, data layers assembled, composite score out, with progressive 
disclosure so every score is decomposable to source data.

================================================================================
SECTION 2: CORE ENDPOINTS
================================================================================

All endpoints return JSON. All accept location as either:
  - address (string, geocoded via Census Geocoder API)
  - lat/lon (float pair)
  - zip_code (string, resolved to constituent tracts)
  - tract_fips (string, 11-digit FIPS code, bypasses geocoding)

When a ZIP code is provided, the API resolves to all intersecting tracts
and returns a population-weighted composite across those tracts.

--------------------------------------------------------------------------------
GET /api/v1/market-fit
--------------------------------------------------------------------------------
Primary endpoint. Returns composite score and all five dimensional sub-scores
with summary-level detail.

Query Parameters:
  - address (str, optional)
  - lat (float, optional)
  - lon (float, optional)
  - zip_code (str, optional)
  - tract_fips (str, optional)
  - radius_miles (float, default=5.0) — defines the market area as all tracts
    with centroids within this radius of the target location
  - provider_tier (str, default="tier1") — which NPI taxonomy tier to use
    for supply calculations. Options: "tier1", "tier1_tier2", "all"
  - weights (object, optional) — override default dimension weights.
    Keys: demand, supply_gap, affordability, employer, competition

--------------------------------------------------------------------------------
GET /api/v1/market-fit/demand
--------------------------------------------------------------------------------
Returns detailed demand-side indicators for the market area.

Response includes:
  - total_population, working_age_population (18-64)
  - age_sex_distribution (5-year cohorts)
  - uninsured_rate, uninsured_count
  - employer_insured_rate (proxy for HDHP exposure)
  - medicaid_rate (not DPC-addressable but context)
  - medicare_rate (not typical DPC but emerging segment)
  - chronic_disease_prevalence:
      - diabetes_pct
      - hypertension_pct
      - obesity_pct
      - copd_pct
      - depression_pct
      - asthma_pct
  - estimated_annual_pcp_visits (based on age-sex utilization rates)
  - ed_visits_acsc_rate (ambulatory care sensitive conditions —
    preventable ED visits that DPC could absorb)
  - svi_composite (overall social vulnerability)

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------

  - pcp_count_tier1_tier2 (expanded definition)
  - pcp_count_all (all potentially primary care providers)
  - pcp_per_100k_population (using selected tier)
  - national_benchmark_pcp_per_100k (for comparison — ~75 per 100k)
  - hpsa_designation: {
      "is_hpsa": true/false,
      "hpsa_score": 0-25,
      "hpsa_type": "geographic" | "population" | "facility",
      "designation_date": "2023-01-15"
    }
  - provider_detail: [
      {
        "npi": "1234567890",
        "name": "Jane Smith MD",
        "taxonomy_code": "207Q00000X",
        "taxonomy_description": "Family Medicine",
        "tier": "tier1",
        "address": "...",
        "lat": 39.10,
        "lon": -94.58,
        "distance_miles": 1.2
      }
    ]
  - fqhc_count (competing free/sliding-scale supply)
  - rural_health_clinic_count
  - urgent_care_count
  - obgyn_count (contextual — women who have a provider relationship)
  - nearest_hospital_miles

--------------------------------------------------------------------------------
GET /api/v1/market-fit/compare
--------------------------------------------------------------------------------
Compare two or more locations side by side.

Query Parameters:
  - locations (list of location objects, max 5)
  - provider_tier (str, default="tier1")

Returns array of market-fit results, one per location, plus a
comparison summary highlighting relative strengths/weaknesses.

GET /api/v1/taxonomy-codes
--------------------------------------------------------------------------------
Returns the full NPI taxonomy configuration with tiering metadata.
Useful for transparency — any consumer of the API can see exactly which
provider types are counted in each tier.

SECTION 3: DATA SOURCES AND RETRIEVAL
================================================================================

Each data source should be implemented as a service class with:
  - A method to fetch data for a given tract FIPS or list of tract FIPS codes
  - Built-in caching (Redis preferred, in-memory fallback)
  - Graceful degradation if a source is unavailable (return nulls, not errors)
  - Data vintage tracking (when was the source data last updated)

Source 1: Census ACS 5-Year Estimates
  API: https://api.census.gov/data/{year}/acs/acs5
  Key tables:
    - B01001 (age/sex)
    - B27001 (health insurance by age)
    - B27010 (health insurance by type — employer, Medicaid, etc.)
    - B19013 (median household income)
    - B15003 (educational attainment)
    - B25070/B25091 (housing cost burden)
    - B23025 (employment status)
  Geography: tract level
  Auth: Free API key from api.census.gov/data/key_signup.html

Source 2: CDC PLACES
  API: https://data.cdc.gov/resource/{dataset-id}.json (Socrata)
  Key measures: DIABETES, BPHIGH, OBESITY, COPD, DEPRESSION, CASTHMA,
    ACCESS2 (no health insurance), CHECKUP (annual checkup), etc.
  Geography: tract level
  Auth: None required (Socrata open data)

Source 3: CDC/ATSDR Social Vulnerability Index (SVI)
  Download: https://www.atsdr.cdc.gov/placeandhealth/svi/data_documentation_download.html
  Fields: RPL_THEMES (overall), RPL_THEME1 (socioeconomic),
    RPL_THEME2 (household/disability), RPL_THEME3 (minority/language),
    RPL_THEME4 (housing/transportation)
  Geography: tract level
  Auth: None (bulk download, load into local DB or file)

Source 4: HRSA HPSA Data
  API: https://data.hrsa.gov/api/
  Also: https://data.hrsa.gov/data/download (bulk CSV)
  Fields: HPSA score, designation type, status, discipline (primary care)
  Geography: varies (geographic, population group, facility)
  Auth: Free API token from data.hrsa.gov

Source 5: NPI Registry (NPPES)
  Bulk download: https://download.cms.gov/nppes/NPI_Files.html
  Monthly full replacement file (~8GB CSV)
  Strategy: Download monthly, filter by taxonomy codes in config,
    geocode practice addresses, load into PostGIS or SQLite with
    spatial index for radius queries.
  Key fields: NPI, provider name, taxonomy code(s), practice address,
    enumeration date, sole proprietor flag
  Note: A provider can have multiple taxonomy codes. Use the primary
    taxonomy (Healthcare Provider Taxonomy Code_1) for classification
    but check all taxonomy fields for Tier 1 codes.

Source 6: County Business Patterns
  API: https://api.census.gov/data/{year}/cbp
  Key tables: county and ZIP-level establishment counts by
    employee size class and NAICS code
  Geography: county and ZIP code (not tract — will need to
    approximate tract-level from ZIP)
  Auth: Same Census API key

Source 7: CMS Provider Data
  API: https://data.cms.gov/provider-data/
  Use for: FQHC locations, hospital locations, ED utilization metrics
  Format: Socrata JSON API
  Auth: None required

Source 8: DPC Practice Registry
  No official federal source. Options:
    - DPC Frontier mapper (https://www.dpcfrontier.com/mapper/)
    - DPC Alliance member directory
    - Hint Health practice directory
    - Manual curation + community contributions
  This is the weakest data layer. Start with manual Kansas City area
  data and build out.

SECTION 4: SCORING METHODOLOGY
================================================================================

Each dimension produces a 0-100 score. The composite is a weighted average
of the five dimensions.

DEFAULT WEIGHTS (configurable per request):
  demand:       0.25
  supply_gap:   0.25
  affordability: 0.20
  employer:     0.20
  competition:  0.10

SCORING APPROACH: Percentile-based normalization against national
tract-level distributions. For each indicator, compute the percentile
rank of the market area's value relative to all US tracts, then combine
within each dimension using indicator weights.

--- DEMAND DIMENSION (weight: 0.25) ---

Indicators and within-dimension weights:
  - uninsured_rate: 0.25 (higher = more demand)
  - chronic_disease_burden: 0.25 (composite of diabetes, hypertension,
    obesity, COPD, depression — higher = more utilization need)
  - working_age_population_density: 0.15 (higher = larger addressable market)
  - ed_acsc_rate: 0.20 (higher = more redirectable demand)
  - svi_socioeconomic_theme: 0.15 (higher vulnerability = more need,
    but tension with affordability)

Scoring direction: For most demand indicators, higher raw values = higher
score, because they signal more unmet need. Exception: cap or penalize
extremely high values (e.g., uninsured > 40%) because at that level
the population may not be able to afford DPC either.

--- SUPPLY GAP DIMENSION (weight: 0.25) ---

Indicators:
  - pcp_per_100k_inverse: 0.35 (fewer PCPs = bigger gap = higher score)
  - hpsa_score: 0.30 (higher HPSA score = more underserved = higher score)
  - nearest_pcp_distance: 0.15 (farther = bigger gap)
  - fqhc_coverage_inverse: 0.20 (fewer FQHCs = less safety net competition)

Note: HPSA designation is binary + scored 0-25. Convert to 0-100 scale.
Non-HPSA areas get the HPSA component scored at 0 but can still score
well on other supply indicators.

--- AFFORDABILITY DIMENSION (weight: 0.20) ---

Indicators:
  - median_household_income: 0.35 (higher = more ability to pay $75-150/mo)
  - dpc_as_pct_of_income: 0.30 (use $100/mo as reference; lower pct = higher score)
  - employment_rate: 0.20 (higher = more stable income)
  - housing_cost_burden_inverse: 0.15 (lower housing burden = more
    discretionary income for DPC membership)

Scoring direction: Higher income / lower cost burden = higher score.
This dimension is in natural tension with the demand dimension — the
populations with the highest need often have the lowest ability to pay.
The composite captures this tension honestly.

--- EMPLOYER OPPORTUNITY DIMENSION (weight: 0.20) ---

Indicators:
  - dpc_target_establishment_density: 0.35 (firms 10-249 employees per capita)
  - low_benefit_industry_pct: 0.30 (higher pct in retail, food service,
    construction, accommodation = more employers motivated to try DPC)
  - median_wage_in_target_firms: 0.20 (mid-range wages optimal — too low
    and employer can't afford DPC; too high and they have good benefits)
  - small_business_growth_rate: 0.15 (if available from CBP year-over-year)

--- COMPETITION/SATURATION DIMENSION (weight: 0.10) ---

Indicators:
  - existing_dpc_count_inverse: 0.40 (fewer existing = less saturated)
  - retail_clinic_density_inverse: 0.20 (fewer = less competition)
  - fqhc_site_density: 0.20 (more FQHCs = more free/low-cost competition)
  - concierge_practice_density: 0.20 (fewer = less premium competition)

Scoring direction: Less competition = higher score. Weight this dimension
lowest because some competition can validate the market.

COMPOSITE CALCULATION:
  composite = (demand * w_d) + (supply_gap * w_s) + (affordability * w_a)
              + (employer * w_e) + (competition * w_c)

SECTION 5: NARRATIVE GENERATION
================================================================================

The /market-fit endpoint includes a "narrative" field — a plain English
summary generated via Claude API with a structured prompt:

Prompt template:

  You are a healthcare market analyst. Given the following DPC Market Fit
  assessment data for {location}, write a 3-4 paragraph narrative summary
  that a physician considering opening a DPC practice or an actuary sizing
  a DPC market would find actionable.

  Data:
  {json_scores_and_indicators}

  Guidelines:
  - Lead with the headline: is this a good market for DPC? Why or why not?
  - Highlight the strongest and weakest dimensions
  - Call out specific tensions between dimensions (high need / low ability to pay)
  - Include 2-3 specific, actionable recommendations
  - Reference specific numbers from the data
  - Keep it under 300 words
  - Do not use bullet points
  - Write in a direct, professional tone

SECTION 7: IMPLEMENTATION NOTES
================================================================================

1. GEOCODING PIPELINE
   - Primary: Census Geocoder API (free, no key needed)
     https://geocoding.geo.census.gov/geocoder/geographies/onelineaddress
   - Returns tract FIPS directly
   - Fallback: lat/lon to tract via FCC API or local shapefile
   - For ZIP codes: use HUD USPS ZIP-Tract crosswalk file

2. RADIUS QUERIES
   - Find all tracts whose centroids fall within radius of target point
   - Use Haversine distance or PostGIS ST_DWithin
   - Population-weight all indicators across included tracts
   - Tract centroid coordinates from Census TIGER/Line

3. NPI DATA MANAGEMENT
   - NPPES monthly file is ~8GB CSV. Strategy:
     a. Download monthly full replacement file
     b. Filter to taxonomy codes in config (all tiers)
     c. Geocode practice addresses (batch via Census Geocoder)
     d. Load into PostGIS table with spatial index
     e. Query: SELECT * FROM providers WHERE ST_DWithin(geom, target, radius)
   - MVP alternative: Use NPPES NPI Registry API for real-time lookups
     (https://npiregistry.cms.hhs.gov/api/) — has rate limits, no spatial query

4. CACHING STRATEGY
   - Census ACS, CDC PLACES, SVI: cache aggressively (annual updates)
   - NPI supply counts: cache per tract, refresh monthly
   - CBP employer data: cache aggressively (annual updates)
   - DPC practice registry: short TTL (most volatile data)
   - Composite scores: cache with key of (tract_set + tier + weights hash)

5. PROGRESSIVE DISCLOSURE
   - /market-fit returns scores + summaries
   - Dimensional detail endpoints return full decomposition
   - Every score traceable: composite -> dimension -> indicator -> raw value -> source
   - Mirrors SDOH API regulatory design: transparency keeps it CDS-exempt

6. ERROR HANDLING
   - If a data source is unavailable, mark dimension as "incomplete"
   - Recalculate composite from available dimensions
   - Never block full response because one source failed
   - Include "data_completeness" field: 1.0 = all sources, <1.0 = partial

7. REUSE FROM SDOH API
